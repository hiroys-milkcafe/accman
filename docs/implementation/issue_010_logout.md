# issue_010: ログアウト機能が明確でない（最優先）

## 問題

ログイン済みの状態でブラウザのアドレスバーからログイン画面へ直接遷移してもセッションが維持されたままになっており、別アカウントへの切り替えや動作確認ができない。

バックエンドには `/logout`（POST）エンドポイントが存在するが、ユーザが明確に操作できるログアウト手段が不十分な可能性がある。

## 状態（Closed）

管理者・一般ユーザともにログアウト動作を確認済み。

## 実装内容

- ヘッダーのヘルプボタン右側にハンバーガーメニュー（☰）を設置し、「ログアウト」項目を追加
- ログイン画面に既ログイン状態でアクセスしたときの挙動: そのままログイン画面を表示（自動ログアウト・リダイレクトなし）
- セッションタイムアウトを設定ファイル `[session]` セクションで管理者・一般ユーザ別に設定可能にし、タイムアウト後は強制ログアウト

実装済みファイル:
- `app/config.py`（`SessionConfig` 追加）
- `config/accman.ini_sample`（`[session]` セクション追加）
- `app/__init__.py`（`before_request` タイムアウトチェック）
- `templates/base.html`（ハンバーガーメニュー追加、インラインログアウト削除）
- `static/css/style.css`（ハンバーガーメニューCSS追加）
