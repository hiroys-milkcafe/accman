# issue_020: CSRF対策の実装

## 優先度

P1（最優先）

## 概要

全POSTエンドポイントにCSRFトークンによる検証が存在しない。ログイン済みの内部ユーザが悪意あるページを踏んだ場合、ブラウザが自動的にセッションCookieを送信するため、PAMエントリ・メールアカウントの改ざん・削除・ログアウトが外部から引き起こされる可能性がある。

## 影響エンドポイント

| エンドポイント | 影響 |
|---|---|
| `POST /logout` | 強制ログアウト |
| `POST /pam/new` | PAMエントリの不正作成 |
| `POST /pam/edit` | PAMエントリの不正改ざん |
| `POST /pam/delete` | PAMエントリの不正削除 |
| `POST /mail/new` | メールアカウントの不正作成 |
| `POST /mail/edit` | メールアカウントの不正改ざん |
| `POST /mail/delete` | メールアカウントの不正削除 |

## ステータス

Closed
