# issue_009: 管理者ログイン画面にID入力欄がない

## 問題

現在の管理者ログイン画面（`/admin/login`）はパスワードのみを入力する設計になっている。
これはパスワードを知っている者であれば誰でも管理者としてログインできることを意味し、
一般ユーザログイン（IDとパスワードの両方が必要）と比べてセキュリティ強度が低い。

設計経緯として「管理者は1人なのでIDの入力は冗長」という判断でID入力欄が省かれたが、
ユーザの要求が繰り返し無視された結果の仕様であり、要件として誤りである。

## 決定

管理者ログインでもIDとパスワードの両方を要求する設計に変更する。

## 未決定事項

入力した管理者IDを何と照合するか、以下のいずれかを選択する必要がある。

1. `admin.bind_dn` の前半部分（例: `cn=admin` の `admin`）を抽出して照合する
2. `admin.bind_dn` と完全一致で照合する
3. 設定ファイルに `admin.id` を別途追加して照合する

## 影響範囲

- `docs/requirements/confirmed.md`
- `docs/design/confirmed_auth.md`
- `app/routes/auth.py`
- `templates/admin/login.html`
