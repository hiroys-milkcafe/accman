# メール管理UI設計 (draft)

## 確定

### 画面一覧

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/mail/` | エントリ一覧 |
| GET | `/mail/new?template=<id>` | 新規作成フォーム |
| POST | `/mail/new` | 新規作成処理 |
| GET | `/mail/edit?dn=<dn>&template=<id>` | 編集フォーム |
| POST | `/mail/edit` | 編集処理 |
| POST | `/mail/delete` | 削除処理 |

### 一覧 `/mail/`

- `scope: mail` のテンプレートをタブで切り替え
- メール BaseDN 以下を LDAP SUBTREE 検索してエントリを一覧表示
- 表示範囲は LDAP の ACL に従う（アプリ側でフィルタしない）

#### コンテナエントリの扱い

SUBTREE 検索の結果には、メールアカウントエントリの上位に位置するコンテナエントリ（`ou=<pamuid>`、`ou=<domain>` 等）も含まれる。これらはメールアカウントとは異なる扱いをする。

- **コンテナの判定**: DNのRDN部分が `{template.container_attr}=` で始まるエントリをコンテナとみなす
  - `container_attr` はテンプレート設定で指定する（デフォルト: `ou`）
- **一覧での表示**: コンテナエントリは「メールアカウントID」列に `{container_attr}=値`（例: `ou=alice`、`ou=example.com`）を表示し、他の列は空欄
- **操作ボタン**: コンテナエントリには「編集」「削除」ボタンを表示しない（全ユーザ共通）
- **コンテナの追加・削除**: 管理者専用の機能。管理画面の設計フェーズで別途対応する（未実装）

### フォームの入力項目

テンプレートの `attributes` 定義に従いフォームを生成する（[PAM管理UI設計](confirmed_pam_ui.md) 参照）。

#### `display_only` 属性の扱い（編集フォーム）

テンプレート属性に `display_only = true` が設定されている場合:

| ユーザ種別 | 表示 |
|------------|------|
| 一般ユーザ | 入力フォームを表示せず、現在の値をテキストで表示のみ。フォーム送信時にも対象属性は含まれない（LDAP には変更を加えない） |
| 管理者 | 通常の編集フォームとして表示・編集可能 |

現在の設定: `mail_user` テンプレートの `mail` 属性に `display_only = true` を設定済み。
理由: 一般ユーザがメールアドレス一覧を自由に変更できると、他ユーザ宛メールの横取りが可能になるリスクがあるため。

### 新規作成処理 `/mail/new` POST

#### 一般ユーザの場合

親DNを以下の手順で自動判定する。

1. フォームの `mail` 属性（最初の値）から `@` 以降をドメインとして取得
2. セッションの `bind_dn` の RDN 値を pamuid として取得（例: `uid=alice,...` → `alice`）
3. 親DN を組み立てる: `{container_attr}={domain},{container_attr}={pamuid},{template.base_dn}`
4. 組み立てた親DN が LDAP 上に存在するか確認
   - 存在する → その DN 配下にエントリを作成
   - 存在しない → エラーを表示して作成させない（管理者にドメインの作成を依頼）

#### 管理者の場合

管理者の新規作成時の親OU選択UIは未実装（管理画面設計フェーズで対応予定）。現状は `template.base_dn` 直下に作成される。

### 編集・削除処理

- 成功 → 一覧にリダイレクト
- 失敗 → エラーメッセージ付きで再表示
