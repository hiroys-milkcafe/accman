# 認証・セッション設計 (確定)

## 確定

### ログインフロー（一般ユーザ） `/login`

1. uid＋パスワードを入力
2. `uid={uid},{pam_base_dn}` を組み立ててLDAP BIND  
   ※ `pam_base_dn` は `[pam] base_dn`。テンプレートごとのエントリ格納先（`[pam] pam_user` 等）とは別に、ログイン専用として存在する（[設定ローダー設計](confirmed_config.md) 参照）
3. 成功 → 一般ユーザセッション生成
4. 失敗 → エラーメッセージ表示、ログイン画面に戻る

### ログインフロー（管理者） `/admin/login`

1. ID＋パスワードを入力
2. 入力IDを設定ファイルの `admin.id` と照合（不一致 → エラー）
3. `admin.bind_dn`＋パスワードでLDAP BIND
4. 成功 → 管理者セッション生成
5. 失敗 → エラーメッセージ表示、ログイン画面に戻る

### ログイン後の画面遷移

- ログイン成功後はトップページへリダイレクト
- トップページにはPAM設定・メール管理へのメニューを表示する

### メニュー構成

| メニュー項目 | 遷移先 | 表示条件 |
|-------------|--------|---------|
| PAM設定 | `/pam/` | `scope: pam` のテンプレートが1件以上存在する |
| メール管理 | `/mail/` | `scope: mail` のテンプレートが1件以上存在する |

### セッション保持情報

| キー | 内容 |
|------|------|
| `bind_dn` | BINDに使用したDN |
| `password` | BINDに使用したパスワード |
| `is_admin` | 管理者か否か（bool） |
| `last_activity` | 最終操作時刻（Unix時刻。タイムアウト判定に使用） |

### ルートガード

- 未ログイン状態でログイン画面以外にアクセスした場合、ログイン画面へリダイレクト

### 認証情報切れハンドリング

- LDAP操作中に `invalidCredentials` が返った場合（セッション保持パスワードが無効になった場合等）、`CredentialExpiredError` が送出される
- アプリ全体の `@app.errorhandler(CredentialExpiredError)` で補足し、セッションを破棄してログイン画面へリダイレクトする
- 管理者セッションは `/admin/login`、一般ユーザセッションは `/login` へリダイレクトする
- リダイレクト時に「認証情報が無効になりました。再度ログインしてください。」をフラッシュメッセージで表示する

### セッションタイムアウト

- 設定ファイルの `[session]` セクションで管理者・一般ユーザ別にタイムアウト秒数を設定する
- すべてのリクエスト（ログイン画面・ログアウト・静的ファイルを除く）で最終操作時刻を更新する
- 最終操作からタイムアウト時間を超えていた場合はセッションを破棄し、管理者は `/admin/login`、一般ユーザは `/login` へリダイレクトする
- ログイン画面に既ログイン状態でアクセスした場合はそのままログイン画面を表示する（タイムアウトチェックのスキップ対象）

### ログアウト操作

全画面のヘッダー右端にハンバーガーメニュー（☰）を設置し、「ログアウト」項目から POST `/logout` を呼び出す。
ハンバーガーメニューにはナビゲーション項目（ホーム・PAM設定・メール管理）もまとめて配置する。

### ルーティング

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/login` | 一般ユーザログイン画面 |
| POST | `/login` | 一般ユーザログイン処理 |
| GET | `/admin/login` | 管理者ログイン画面 |
| POST | `/admin/login` | 管理者ログイン処理 |
| GET | `/` | トップページ（メニュー） |
| POST | `/logout` | ログアウト・セッション破棄 |
