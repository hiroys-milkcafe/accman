# ヘルプドロワー設計 (確定)

## 概要

全画面共通のヘルプドロワー。ヘッダーのヘルプボタン押下で画面右からスライド表示する。

## UI構造

```
[ヘッダー: accman | ヘルプ | ☰]
[操作画面（ドロワー分だけ幅が縮む）         ][ヘルプドロワー]
```

ハンバーガーメニュー（☰）の内容:
```
ホーム
PAM設定（pam テンプレートが1件以上ある場合）
メール管理（mail テンプレートが1件以上ある場合）
─────────
ログアウト
```

## 動作仕様

| 項目 | 仕様 |
|------|------|
| 表示トリガー | ヘッダーの「ヘルプ」ボタン |
| 閉じるトリガー | ドロワー内の ✕ ボタンのみ |
| ドロワー外クリック | 閉じない（ヘルプを見ながら操作できるようにするため） |
| 操作画面への影響 | ドロワー表示中は `body` に `padding-right: 400px` を付与し、操作画面の表示幅を縮める。ドロワーは操作画面にかぶさらない |
| アニメーション | ドロワーの `right` と `body` の `padding-right` を `.25s ease` でトランジション |
| ドロワー幅 | 400px |
| 表示位置 | `position: fixed`（スクロールに追従しない） |

## ヘルプ内容

- 各テンプレート（`base.html` の `{% block manual %}`）で画面ごとに定義する
- `{% if is_admin %}` でログインユーザの種別に応じた説明を切り替える

### 対象画面と内容

| 画面 | 管理者向け内容 | 一般ユーザ向け内容 |
|------|---------------|------------------|
| ログイン（一般） | — | ユーザID・パスワードの説明、管理者ログインへの案内 |
| ログイン（管理者） | IDとパスワード入力の説明、一般ユーザログインへの案内 | — |
| メニュー | 管理者として全操作可能である旨 | ACLによる制限がある旨 |
| PAM設定一覧 | 全エントリ操作可能な旨 | ACLによる制限がある旨 |
| PAM 新規作成 | 入力規則・各項目の説明 | 同左 |
| PAM 編集 | 入力規則・パスワード変更手順・各項目の説明 | 同左 |
| メール管理一覧 | 全エントリ操作可能な旨 | ACLによる制限がある旨 |
| メール 新規作成 | 入力規則・各項目の説明 | 同左 |
| メール 編集 | 入力規則・パスワード変更手順・各項目の説明 | 同左 |

## 実装箇所

| ファイル | 変更内容 |
|----------|---------|
| `templates/base.html` | ドロワーHTML・ヘルプボタン・開閉JS |
| `static/css/style.css` | ドロワーCSS・`body.manual-open` のpadding |
| `templates/*/` 各テンプレート | `{% block manual %}` でヘルプ内容を定義 |
