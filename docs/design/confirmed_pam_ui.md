# PAM管理UI設計 (draft)

## 確定

### 画面一覧

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/pam/` | エントリ一覧 |
| GET | `/pam/new?template=<id>` | 新規作成フォーム |
| POST | `/pam/new` | 新規作成処理 |
| GET | `/pam/edit?dn=<dn>&template=<id>` | 編集フォーム |
| POST | `/pam/edit` | 編集処理 |
| POST | `/pam/delete` | 削除処理 |

### 一覧 `/pam/`

- `scope: pam` のテンプレートをタブで切り替え（例: PAMユーザ・PAMグループ・Sudoer）
- PAM BaseDN以下をLDAP検索してエントリを一覧表示
- 各行に「編集」「削除」リンク
- 表示範囲はLDAPのACLに従う（アプリ側でフィルタしない）

### フォームの入力項目

テンプレートの `attributes` 定義に従って以下のルールでフォームを生成する。

| テンプレート属性 | フォームへの反映 |
|-----------------|----------------|
| `label` | フィールドのラベル |
| `type: text` | テキスト入力。編集時は現在値を表示する |
| `type: password` | **新規作成時**: パスワード入力欄（必須）。**編集時**: インライン入力欄は設けず「パスワード変更」ボタンを表示。ボタン押下でモーダルを開き、入力確定後にhidden inputへセットしてPOSTに含める。ボタンを押さずに保存した場合はパスワードを変更しない |
| `type: number` | 数値入力。編集時は現在値を表示する |
| `required: true` | 必須マークを付けバリデーションをかける |
| `multi: true` | 値を複数入力できる行を追加するUIを提供する。編集時は現在値を各行に展開して表示する |
| `required: false`（編集時） | 空欄送信で属性削除（MODIFY_REPLACE 空リスト） |
| `required: true`（編集時） | 空欄送信はバリデーションエラー。保存しない |

### 新規作成処理 `/pam/new` POST

- RDNはテンプレートの `rdn_attr` の値から生成（親DN: PAM BaseDN）
- 成功 → 一覧にリダイレクト
- 失敗 → エラーメッセージ付きでフォーム再表示

### 編集・削除処理

- 成功 → 一覧にリダイレクト
- 失敗 → エラーメッセージ付きで再表示
